2) Front Messenger - Conversation Scroll + Composer Input
----------------------------------------------------------
Files changed:
- templates/front/messenger/messages.html.twig
- templates/front/messenger/_messenger_styles.html.twig
- templates/front/messenger/_messenger_scripts.html.twig
- templates/front/messenger/_message_input.html.twig

What was fixed:
- Removed inline hardcoded chat area sizing from messages.html.twig.
- Made chat panel structurally scrollable with CSS.
  - chat-view/chat-view-inner/chat-main sizing and overflow constraints.
  - chat-messages set as the real scroll container (overflow-y auto).
  - Added max-height fallback for chat-messages.
- Added JS fallback (in case stale CSS is cached):
  - Forces chatMessages overflowY=auto, overflowX=hidden, and maxHeight.
  - Keeps initial scroll-to-bottom behavior.
- Removed conflicting scrollIntoView() on input that could disrupt internal chat scrolling.
- Hid native file input controls robustly:
  - CSS hard-hide for file inputs.
  - Added hidden attribute on attachments inputs in _message_input.html.twig.
- Normalized paperclip button shape/layout:
  - fixed width/min-width/flex basis/padding/box-sizing for circular button rendering.

Result:
- Conversation panel can scroll to older messages.
- Native file input no longer breaks composer layout.


3) Front Messenger - Text + Image Side Alignment
-------------------------------------------------
Files changed:
- templates/front/messenger/_message_item.html.twig
- templates/front/messenger/_messenger_styles.html.twig

What was fixed:
- Grouped message text + attachments under one wrapper in markup:
  - Added .message-body in _message_item.html.twig.
- Enforced sender-side pinning rules in CSS:
  - Sent (.you) content/media blocks pinned to the right.

  - .message-content
  - .message-attachments
  - .message-attachment-image

- Adjusted image sizing constraints to reduce layout drift:
  - max-width/max-height limits and controlled width behavior.

Result:
- Text and images are consistently pinned by sender side (right for own messages, left for others).
